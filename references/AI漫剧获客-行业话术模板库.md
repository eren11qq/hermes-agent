# AI漫剧获客 — 行业话术模板库

> 用途：outreach-generator Skill 调用，每个模板通过 `{变量}` 填入具体线索信息
> 原则：每句话要么展示行业洞察，要么指向对方具体痛点——不说废话，不套模板感

---

## 一、话术核心原则（5条铁律）

```
1. 第一句必须提到对方的具体作品/融资/招聘动态
   → 证明你不是群发，你真的研究过他

2. 痛点不说"效率低"，说具体数字
   → "角色一致性差"不如"第3集和第5集主角脸都不一样了"

3. 价值主张只说一件事
   → 一个模板只打一个痛点，不要罗列功能

4. 结尾给低门槛行动
   → 不是"安排演示"，是"方便聊15分钟吗"

5. 每一次跟进必须携带新信息
   → 不是"checking in"，是新案例/新数据/新角度
```

### 渠道一：脉脉/LinkedIn 私信（首选）

**特点**：字数受限、对方看profile、可以附带个人背景
**关键**：前30字决定是否点开

#### 模板 A：融资型企业（HOT Lead）

## 三、按客户类型分类

### 类型一：内容制作公司（如灵境AI、星迹互动、灵漫快创）

```
特征：有钱（融资）、有产量（月产数百部）、有痛点（产能天花板）
决策人：创始人/CEO 或 技术VP
核心痛点排序：产能 > 质量一致性 > 出海 > 成本

切入话术框架：
1. 赞扬具体作品（证明你做了功课）
2. 指出产能天花板（300部→1000部不可能靠堆人）
3. 给出一个可执行方案（不是卖产品，是解决问题）
4. 用同行案例证明（不是吹牛，是事实）
```

#### 模板 I：内容制作公司 — 产能切入

```
变量：{founder_name} {company_name} {hit_work} {view_count} {monthly_output} {team_size}

---

{founder_name}老师您好，

《{hit_work}》{view_count}播放，这个成绩在AI漫剧赛道非常难得。
更不容易的是——{monthly_output}部的月产量，{team_size}人团队能做到这个工业化效率。

不过{team_size}人撑{monthly_output}部，人均产能应该快到天花板了。
再多招{estimated_new_hires}个人也未必解决问题——新人上手要时间，管理成本会指数增长。

我们{your_product}帮一家月产{reference_output}部的团队做过一件事：
把{key_process}环节的耗时从{old_time}压缩到了{new_time}，不增加人手的情况下产能提升了{gain}%。

实现方式很简单：{one_sentence_how}。

如果{company_name}也想突破产能天花板，可以聊聊。

{your_name} | {your_product} | {your_contact}
```

#### 模板 J：内容制作公司 — 质量一致性与爆款复制

```
变量：{founder_name} {hit_work} {monthly_output}

---

{founder_name}老师，

《{hit_work}》爆了之后，最大的挑战变成了：下一部怎么复制？

{monthly_output}部的月产量，但凡有一部角色崩坏、画风跳变，用户的信任就没了。
而且AI漫剧观众对"画风不对"特别敏感——从评论就能看出来。

我们在{your_product}里解决这个问题的方法是：{solution_brief}。
帮{reference_customer}把角色一致性投诉从{before}降到了{after}，同时保持了{monthly_output}的产出量。

有时间聊聊吗？

{your_name} | {your_product} | {your_contact}
```

## 四、六大场景反拒话术

### 场景1："我们已经有工具了"

```
理解。{company_name}现在的工具链应该覆盖了{likely_coverage}。

不过我们接触的很多团队发现一个共同问题：
通用工具在{niche_scenario}这个细分场景上，效率和专业工具差{performance_gap}。

就像做菜——菜刀也能切，但切丝还是擦丝器快。

要不您花5分钟看看我们在{niche_scenario}上的做法？
不替代现有工具，补一个短板。
```

### 场景2："太忙了，没时间"

```
完全理解，{monthly_output}部的产量节奏确实没时间。

这样——我不占用您太多时间。只问一个问题：
{one_specific_question_about_their_pain}

如果您觉得这个问题值得解决，我们再聊。不值得，我绝不再打扰。
```

### 场景3："我们自己能做"

```
{company_name}的技术团队确实有这个实力——{specific_technical_achievement}就证明了。

不过自研有个隐性成本值得算一下：
{self_build_cost_breakdown}。

{reference_customer}之前也走自研，花{self_build_time}做了第一版，
后来发现维护成本太高，转而用我们的方案。不是技术问题，是ROI问题。

当然如果{company_name}评估下来自研ROI更高，那您选自研完全合理。
如果愿意，我可以把我们的方案和自研成本做个对比，帮您做决策更清晰。
```

### 场景4："预算不够"

```
理解。不过换个角度看——

如果{your_product}帮{company_name}把{key_metric}提升{gain}%，
那每个月的额外产出就是{extra_output_value}。

相比这个增量，{your_price}的投入相当于{payback_period}就回本了。

当然如果您觉得现在不是时机，我们可以过{follow_up_period}再聊。
行业变化快，说不定到时有新的痛点需要我们帮您解决。
```

### 场景5："要跟团队商量"

```
没问题，这是应该的。

为了方便您内部讨论，我整理几句话方便您转述：

"{your_product}帮我们解决{one_line_problem}。
  {reference_customer}用了之后{one_line_result}。
  投入{one_line_cost}，预计{one_line_roi}回本。"

另外我可以配合做一次团队内部分享或demo——您内部推的时候更有说服力。

下周这个时间我跟您确认一下进展？
```

### 场景6：无回复（二次触达）

```
变量：{contact_name} {days_since_last} {new_insight_or_data}

---

{contact_name}您好，上次联系是{days_since_last}天前，冒昧再发一条——

最近注意到一个新变化：{new_insight_or_data}。
这和{company_name}在做的事直接相关，因为：{why_relevant}。

我们在帮{reference_customer}处理类似问题时有一个做法，可能对您也有参考价值。

这条消息您方便时看一眼就行，不用着急回。
如果这个话题不是您现在优先关注的方向，也完全理解。

{your_name}
```

## 五、跟进节奏编排（7日触达链）

```
Day 1（周二上午 10:00）:
  第一封邮件/第一条私信 — 开头提到具体信息

Day 3（周四下午 14:30）:
  二次触达——用"场景2反拒话术"
  不是"checking in"，带新数据或行业新闻

Day 7（下周二上午 10:00）:
  最后一次主动触达——用"场景6无回复话术"
  给台阶下："不是您优先方向也完全理解"

Day 14+:
  转入培育队列
  每2周发一次行业洞察简报（不是产品推销）
  当对方有融资/招聘/新产品动态时重新激活
```

## 七、自检清单

每次写完触达文案后，用这 6 条检查：

```
□ 第一句是否提到了对方的具体信息？（作品/融资/招聘——不能用通用词）
□ 是否只说了一个痛点？（不是罗列功能，是解决一个具体问题）
□ 是否有具体数字？（效率提升不是"显著"，是"3倍"）
□ 是否给了低门槛行动？（15分钟聊一下 > 安排正式演示）
□ 读起来像不像群发？（找人试读——如果觉得"这条可以发给任何人"那就重写）
□ 是否给了台阶下？（对方不感兴趣也能体面拒绝，不会撕破脸）
```
