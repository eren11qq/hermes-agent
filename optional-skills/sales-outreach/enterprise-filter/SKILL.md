# 企业 vs 个人过滤器 (Enterprise Filter)

## 定位

本 Skill 是工具型 Skill，不直接面向用户。被 `lead-finder` 和 `company-researcher` 在内部调用，判断一个社媒账号背后是企业/团队还是个人创作者。

## When to Use

- lead-finder 从 B站/小红书/抖音/快手 搜到账号后，需要判断是否进入销售管道
- company-researcher 调研前，需要确认目标是否值得深度调研
- 线索入库 CordysCRM 前，需要打上 `is_enterprise` 标签
- 销售团队要求对现有线索批量重审

## When NOT to Use

- 已有明确工商注册信息的公司——直接用 ENScan_GO 查，不需要推理
- 海外平台账号（YouTube/Twitter/LinkedIn）——信号体系不同，需另行设计
- 用户手动标记过的账号——人工标签优先级最高

## ENScan_GO 集成（硬查优先）

```
输入线索 → 提取公司名关键词 → ENScan_GO 查询
  ├── 查到工商注册 → 跳过信号计分，直接 ENTERPRISE ✅
  ├── 查不到 → 进入 signal-scoring / bant / icp 模式
  └── 公司名模糊 → 列出候选人 → Claude 选最匹配 → 再查
```

注意：很多 AI 漫剧工作室未注册工商（个人工作室形式），ENScan_GO 查不到不代表不是企业。此时社媒信号判断更重要。

---

## 三种过滤模式

| 模式 | 核心逻辑 | 适用阶段 |
|------|---------|---------|
| **signal-scoring** (默认) | 20+ 个信号加权计分 | 冷启动期 |
| **bant** | BANT 四维资格评估 | 有销售团队后 |
| **icp** | 理想客户画像匹配 | 规模化运营期 |

调用方通过参数选择模式。默认 `signal-scoring`。

## 模式一：信号计分 (signal-scoring) —— 默认

### 核心 Prompt

```
你是 B2B 销售资格审查专家，专门评估 AI 漫画/动画/视频领域的社交媒体账号。

分析以下账号主页信息，判断运营者是"企业/团队"还是"个人创作者"。

## 加分信号（企业/团队特征）

### 组织信号
- [+3] 简介含"工作室""团队""公司""传媒""文化传播""影业""动漫"
- [+3] 蓝V/企业认证标识
- [+2] 简介含"商务合作""接单""定制"且有企业邮箱
- [+2] 发布过招聘信息（招画师/编剧/剪辑/技术）

### 规模信号
- [+2] 多账号矩阵（主号+子号+员工号+IP号）
- [+2] 视频中出现办公室/工位/会议室（非卧室/咖啡厅）
- [+1] 简介列了多个成员或分工（编剧/美术/后期/配音）
- [+1] 有公司官网链接

### 内容信号
- [+2] 内容有统一品牌包装（片头/LOGO/固定片尾）
- [+2] 日更或多平台同步更新（工业化产出节奏）
- [+1] 内容类型多样（漫剧+短视频+教程+商单案例）

### 商业信号
- [+3] 展示商单案例或有品牌合作内容
- [+2] 使用付费专业工具（ComfyUI企业版/SD商用授权/AE正版）
- [+1] 参加行业展会/比赛/平台官方活动
- [+1] 有融资/获奖/媒体报道

## 减分信号（个人创作者特征）

### 身份信号
- [-3] 简介用"个人""UP主""博主""自由职业""独立作者"
- [-2] 只有单一社交媒体账号，无其他平台链接
- [-2] 使用个人邮箱（@qq.com/@gmail.com/@163.com），非企业域名

### 规模信号
- [-2] 所有内容一个人出镜/配音/署名
- [-2] 在卧室/家里录制，无办公环境
- [-1] 简介无任何团队描述

### 内容信号
- [-1] 更新不规律，经常断更 2 周以上
- [-1] 内容以教程/个人分享为主，非商业产出
- [-1] 只有单一内容类型

### 商业信号
- [-3] 明确表示"不接商单""爱好而已"
- [-2] 只用免费工具（可灵免费版/剪映免费/Midjourney试用）
- [-1] 内容无任何商业元素（无广告/无合作/无付费内容）
- [-1] 粉丝量明显低（< 1000）且无增长趋势

## 判定规则
- 总分 >= 5: ENTERPRISE — 进入销售管道
- 总分 2~4: UNCERTAIN — 标记需人工复核，暂入 Warm 管道
- 总分 < 2: INDIVIDUAL — 自动过滤，不入库

## 输出格式
{
  "verdict": "ENTERPRISE | UNCERTAIN | INDIVIDUAL",
  "score": 数字,
  "signals_found": [
    {"signal": "信号描述", "weight": +2, "evidence": "从简介/内容中提取的具体证据"}
  ],
  "signals_missed": ["本应出现但未出现的信号"],
  "confidence": "HIGH | MEDIUM | LOW",
  "reasoning": "一句话总结判断依据",
  "recommended_mode": "如果 confidence=LOW，建议切换到 bant 或 icp 模式"
}
```

### 执行步骤

1. **收集信息源**：读取目标账号的以下页面
   - 社媒主页（简介、认证状态、粉丝数）
   - 最近 10 条内容（标题、标签、评论互动）
   - 如有官网链接 → Jina Reader 抓取
   - 如有招聘信息 → 记录岗位数量和类型

2. **并行走信号检测**：对照上述 20+ 信号逐一检查，记录证据

3. **计算总分 + 输出判定**

4. **UNCERTAIN 处理**：
   - 自动触发 `company-researcher` 补充调研
   - 或加入人工复核队列

## 模式二：BANT 资格评估 (bant)

### 核心 Prompt

```
你是 B2B 销售资格分析师。用 BANT 框架评估以下 AI 漫剧/视频领域的潜在客户。

## B — Budget（预算/付费能力）[1-5分]

评估线索是否有能力和意愿为专业 AI 创作软件付费：

5分: 有明确预算信号——在招 AI 岗位、展示商单（报价>=10万）、有融资记录
4分: 正在使用付费工具（SD商用/ComfyUI企业版/AE正版）、有持续商单
3分: 有商业变现但不透明——接单但未公开报价、有广告但不频繁
2分: 偶尔接小单、用免费工具为主
1分: 无任何商业变现、明确"爱好"定位、只用免费工具

证据来源: 商单内容、招聘信息、融资新闻、工具使用痕迹

## A — Authority（决策权）[1-5分]

评估决策链路和采购权限：

5分: 创始人/CEO 账号，简介明确"创始人"，可直接决策
4分: 技术负责人/内容总监账号，对工具选型有决定权
3分: 团队账号（非个人），需找对联系人但有团队支撑
2分: 单人运营但有商业意识——自己决定但预算有限
1分: 单人、无商业意识、无决策场景——不存在"采购"这个动作

证据来源: 简介职位、团队构成、联系方式类型

## N — Need（需求深度）[1-5分]

评估痛点与产品的匹配度：

5分: 痛点直接命中——产能瓶颈明确（想日更做不了）、一致性差导致掉粉、在公开抱怨工具
4分: 痛点可推断——内容产出量大（日更多平台）、质量要求高但工具原始（手动AE）
3分: 有相关需求但未明确表达——做 AI 内容但量不大、工具够用
2分: 轻度需求——偶尔做 AI 内容、对效率不敏感
1分: 无匹配需求——不做 AI 漫剧/动画/视频内容

证据来源: 内容频率、工具使用、公开言论、评论区反馈

## T — Timeline（紧迫性）[1-5分]

评估采购紧迫程度：

5分: 极度紧迫——正在招 AI 岗、刚融资催促扩张、内容停更找工具
4分: 比较紧迫——公开抱怨工具、竞品已升级、内容质量明显下滑
3分: 有计划但不急——在关注工具但还没开始评估
2分: 远期需求——"将来可能"但没有明确时间表
1分: 无紧迫感——对现状满意、无改进意愿

证据来源: 招聘动态、融资时间、内容趋势、公开言论

## 四维加权

BANT 总分 = B×0.3 + A×0.2 + N×0.3 + T×0.2

（Budget 和 Need 权重最高——有钱+有痛才有交易）

## 判定规则
- 总分 >= 4.0: HOT — 本周内触达
- 总分 2.5~3.9: WARM — 培育，2周后复查
- 总分 < 2.5: COLD — 过滤或长期培育

## 输出格式
{
  "verdict": "HOT | WARM | COLD",
  "bant_scores": {"B": 4, "A": 3, "N": 5, "T": 4},
  "bant_total": 4.1,
  "weakest_dimension": "Authority",
  "weakest_reason": "团队账号但未找到明确决策人",
  "reasoning": "痛点明确(产能瓶颈)+有预算信号(在招AI岗)，但需确认决策人",
  "next_action": "查找决策人联系方式 → 发送个性化触达"
}
```

## 模式三：ICP 画像匹配 (icp)

### ICP 画像定义 v1.0

```
AI 漫剧软件 ICP v1.0

## 必须满足（硬性条件，不满足直接 REJECTED）

- 内容领域: AI 漫剧 / AI 动画 / AI 视频 / 动态漫（四选一或多选）
- 团队规模: >= 3 人（有分工合作，不是单人包揽全流程）
- 内容产出: >= 每周 3 条（有量产需求，工业化才有意义）
- 工具使用: 使用 AI 生成工具（SD/MJ/ComfyUI/可灵/即梦等）
- 排除: 纯传统手绘动画（不用 AI）、实拍为主偶尔 AI、非商业内容

## 加分维度

### 内容管线复杂度 [权重: 25%]
- 多平台发布（B站+小红书+抖音至少2个）
- 多种内容类型（正片+花絮+教程+预告）
- 高频更新（日更 > 周更 > 不定期）

### 技术成熟度 [权重: 20%]
- 使用专业工具链（ComfyUI工作流 + AE/PR后期）
- 自训练模型（LoRA/Checkpoint）
- 有技术分享内容（说明有深度使用）

### 商业规模 [权重: 25%]
- 商单频率：月均 >= 2 条商单
- 粉丝规模：B站 >= 10万 or 抖音 >= 50万 or 小红书 >= 5万
- IP变现：有自有IP、周边、付费内容

### 采购动机 [权重: 20%]
- 在招 AI/技术/后期岗位
- 公开抱怨工具（产能/一致性/效率）
- 关注竞品或同类工具
- 有融资/扩张新闻

### 组织成熟度 [权重: 10%]
- 有公司官网/企业邮箱
- 参加行业活动/展会
- 有媒体报道/获奖记录
```

### 核心 Prompt

```
你是 AI 漫剧软件 ICP 匹配引擎。将以下线索与 ICP v1.0 进行逐维比对。

## 执行步骤

1. 硬性条件检查 — 先看 5 个"必须满足"条件
   - 任一不满足 → 直接输出 REJECTED + 不满足的具体条件

2. 加分维度打分（每维 1-5 分）

3. 加权计算:
   ICP 匹配度 = 内容管线×0.25 + 技术成熟度×0.20 + 商业规模×0.25 + 采购动机×0.20 + 组织成熟度×0.10
   满分 5.0

4. 排除规则二次检查:
   - 单人创作、无协作分工
   - 内容产出 < 每周 2 条
   - 只用手机 APP / 免费工具
   - 纯兴趣导向、无商业变现
   - 内容不在 AI 漫剧/动画/视频领域

5. 输出判定:
   - 匹配度 >= 80% (4.0): ENTERPRISE — 高优先级进入管道
   - 匹配度 50-79% (2.5-3.9): UNCERTAIN — 补充调研
   - 匹配度 < 50% (< 2.5): INDIVIDUAL — 过滤

## 输出格式
{
  "verdict": "ENTERPRISE | UNCERTAIN | INDIVIDUAL | REJECTED",
  "icp_match_pct": 85,
  "dimension_scores": {
    "content_pipeline": 4,
    "tech_maturity": 5,
    "business_scale": 4,
    "purchase_motivation": 5,
    "org_maturity": 3
  },
  "hard_conditions": {
    "content_domain": true,
    "team_size_min3": true,
    "weekly_output_min3": true,
    "uses_ai_tools": true,
    "non_commercial_excluded": true
  },
  "exclusion_hit": null,
  "top_strength": "采购动机强——正在招AI岗+公开抱怨产能",
  "top_weakness": "组织成熟度低——无官网无企业邮箱",
  "next_action": "推进管道 → company-researcher 深度调研"
}
```

## 模式选择指南

```
冷启动期（第 1-2 周）:
  → 默认 signal-scoring
  → ENScan_GO 作为硬证据优先
  → 目标: 快速过滤，不错杀疑似企业

销售团队到位后（第 3-4 周）:
  → signal-scoring 初筛 + bant 深度评估 Hot Leads
  → 销售总监看到 BANT 报告能直接分配跟进
  → 目标: 输出的 Hot Lead 有明确跟进理由

规模化运营（第 2 个月+）:
  → icp 为主
  → 根据成交客户数据反向校准 ICP 权重
  → 目标: 精准匹配，减少销售在低质量线索上浪费时间
```

## 验证

### 自测方法

取 20 个已知标签的账号（10 个企业 + 10 个个人），跑过滤器，计算：

- 准确率 = 正确判定数 / 总数（目标 >= 85%）
- 召回率 = 正确识别的企业数 / 实际企业数（目标 >= 90%，宁可误收不错杀）
- 误杀率 = 错误判定为个人的企业数 / 实际企业数（目标 < 5%）

### 持续优化

1. 每周抽查 10 条判定结果
2. 对比实际成交/流失的客户，回看当初的过滤器打分
3. 调整信号权重和阈值
4. 更新 ICP 画像（v1.0 → v1.1 → ...）
