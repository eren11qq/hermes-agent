# 公司深度研究 (Company Researcher)

## When to Use

当销售管道中存在需要深度验证和评估的潜在客户公司时，触发此技能。具体触发场景包括：

- 从线索采集阶段收到了新的公司名单，需要逐一评估其销售价值
- 需要在联系客户之前了解其内容生产流程、团队规模和痛点
- 需要验证公司的合法性和经营状态（工商注册、ICP 备案等）
- 需要找到目标公司的关键决策人及其联系方式
- 需要对线索进行 HOT / WARM / COLD 分级

## When NOT to Use

以下情况不应触发此技能：

- 公司信息已经过完整研究且未超过 30 天（直接使用已有报告）
- 目标为个人创作者且粉丝量小于 1 万（冷线索，不值得深度研究）
- 目标公司已被明确标记为"不合作"或"已流失"
- 仅需要快速查询工商基本信息（使用更轻量的工具即可）

## Tools Required

本技能依赖以下 MCP 工具和外部命令：

| 工具 | 用途 | 来源 |
|------|------|------|
| `enscan-go` (MCP) | 企业工商注册、ICP 备案、App 信息、招聘数据查询 | ENScan_GO MCP Server |
| `r.jina.ai` | 将任意网页转换为 Markdown 格式阅读 | Jina Reader (HTTP) |
| `passive-recon` (MCP) | 多数据源 OSINT 信息收集（15+ 数据源） | passive-recon MCP Server |
| `exa-search` (MCP) | 网页搜索，获取额外信号 | Exa Search MCP Server |
| `web-fetch` | 直接获取网页内容 | Claude 内置工具 |
| `web-search` | 常规网页搜索 | Claude 内置工具 |

## Procedure

按以下步骤逐一执行，每个步骤的结果记录在对应的分析区块中：

### 第1步：打开官方网站与社交媒体主页

```
1. 使用 web-fetch 或 r.jina.ai 打开公司官网首页
   curl -s "https://r.jina.ai/http://公司官网URL" -H "Accept: text/markdown"
   
2. 使用 web-search 搜索公司在各大平台的官方账号：
   - Bilibili: site:bilibili.com "公司名称"
   - 抖音: site:douyin.com "公司名称"
   - 小红书: site:xiaohongshu.com "公司名称"
   - 微信公众号: 微信搜一搜

3. 记录以下信息：
   - 内容类型（漫画/视频/动画/混合）
   - 更新频率（日更/周更/不定期 → 是否存在产能压力？）
   - 内容质量（是否有明显 AI 痕迹？→ 是否需要更好的工具？）
```

### 第2步：分析内容生产流程

```
1. 从官网、招聘信息、社交媒体描述中推断当前工具链：
   - 生成工具：SD? Midjourney? ComfyUI?
   - 动画工具：After Effects? 自研?
   - 配音工具：AI TTS? 人工?

2. 搜索招聘信息推断团队规模：
   - 使用 enscan-go 搜索公司招聘数据
   - 使用 web-search 搜索 "公司名 招聘" 查看 Boss 直聘、拉勾等平台
   - 使用 web-fetch 查看公司官网的"关于我们"或"团队介绍"页面

3. 识别效率瓶颈：
   - 生成速度？一致性？后期制作？
```

### 第3步：发现痛点（关键步骤）

```
1. 使用 web-search 搜索负面反馈：
   "公司名 吐槽" "公司名 差评" "公司名 问题"

2. 使用 web-fetch 查看社交媒体评论区：
   重点观察观众对以下方面的抱怨：
   - 角色一致性
   - 画面质量
   - 更新速度
   - 剧情质量

3. 从招聘信息反推痛点：
   使用 enscan-go 查询招聘岗位，分析：
   - 正在招聘 "AI 技术总监" → 寻求更好的 AI 解决方案
   - 正在招聘 "后期制作" → 后期产能不足
   - 正在招聘 "角色设计师" → 角色一致性有问题

4. 使用 passive-recon 获取更多 OSINT 信号：
   - GitHub 开源项目参与度
   - 技术博客 / 公众号文章
   - 行业会议演讲
```

### 第4步：企业合法性验证

```
1. 使用 enscan-go (MCP mode) 查询企业工商信息：
   enscan-go search --name "公司全称" --mode enterprise

2. 交叉验证：
   - ICP 备案号是否匹配官网
   - 经营状态是否为"存续"
   - 是否有经营异常或行政处罚记录

3. 使用 exa-search 搜索其他信号：
   搜索 "公司名 融资" "公司名 投资" 判断资本实力
```

### 第5步：判断真实需求等级

根据收集到的信息，使用以下评分模型进行打分：

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 痛点清晰度 | 30% | 1-10 分：有明确公开抱怨=8+，仅推断=4-6，无信号=1-3 |
| 预算信号 | 30% | 1-10 分：已融资/大团队/主动寻找付费方案=8+，中型团队=4-7，小团队=1-3 |
| 紧迫程度 | 25% | 1-10 分：正在招聘相关岗位/公开表示产能不足=8+，可推断=4-7，无信号=1-3 |
| 团队规模倍率 | 15% | 100人+=1.5x，30-100人=1.2x，10-30人=1.0x，<10人=0.7x |

最终得分 = (痛点×0.3 + 预算×0.3 + 紧迫×0.25) × 团队规模倍率

分级标准：
- **HOT (70+)**: 明确痛点 + 有预算 + 正在积极寻找解决方案
- **WARM (40-69)**: 痛点存在但不紧迫 + 中小团队
- **COLD (<40)**: 个人创作者 + 免费工具即可满足需求

### 第6步：识别决策人

```
1. 使用 web-search 搜索：
   "公司名 CEO" "公司名 创始人" "公司名 技术总监"

2. 使用 web-fetch 查看 LinkedIn：
   site:linkedin.com "公司名"

3. 使用 passive-recon 收集联系方式：
   - 邮箱格式推断（从域名 + 常见格式）
   - 社交媒体 DM 可行性评估

4. 记录最佳联系方式优先级：LinkedIn DM > 邮箱 > 微博私信
```

## Output Format

研究完成后，必须按以下格式输出报告：

```
🏢 公司：XX AI 漫画工作室
🔗 来源：Bilibili | 50万粉丝
👥 团队：约30人（从招聘信息推断）
📊 内容产出：3部动态漫画/天

🛠 当前流程分析
├── 生成：Stable Diffusion + LoRA（自定义角色模型）
├── 动画：After Effects 手动关键帧
├── 配音：AI TTS（未知方案）
└── 痛点：① AE 手动工作量大 ② 角色一致性差 ③ 产出速度慢

🔍 真实需求评估
├── 官网称"每天只能做3集" → 产能瓶颈
├── 评论区抱怨"角色脸变来变去" → 一致性问题
├── 正在招聘"AI 技术总监" → 正在寻求更好的解决方案
└── 结论：HOT LEAD — 正在寻找提升产能和一致性的工具

📞 决策人
├── CEO：张三 (linkedin.com/in/zhangsan)
├── 技术负责人：李四 (lisi@company.com)
└── 最佳联系方式：LinkedIn DM 或邮箱

📈 评分明细
├── 痛点清晰度：8/10
├── 预算信号：7/10
├── 紧迫程度：8/10
├── 团队规模倍率：1.0x（30人）
└── 最终得分：75 → HOT LEAD
```

## Verification

研究完成后，逐项自检：

- [ ] 官网和至少一个社交媒体主页已访问并记录
- [ ] 内容类型、更新频率、质量评估已完成
- [ ] 当前工具链已推断（至少识别生成工具和后期工具）
- [ ] 团队规模已有估算依据（招聘信息或公开数据）
- [ ] 至少找到 1 个具体痛点信号（评论/招聘/公开言论）
- [ ] 工商注册信息已通过 `enscan-go` 验证
- [ ] ICP 备案与官网一致性已验证
- [ ] 评分模型各维度均已打分并有依据
- [ ] 最终等级已判定（HOT/WARM/COLD）
- [ ] 至少找到 1 名决策人及其联系方式
- [ ] 报告已按 Output Format 格式输出

如任一项未完成，返回对应步骤补充研究。

## Example

**执行过程：**

```

# → 获取官网内容，确认：动态漫画工作室，日更2-3部

web-search "星辰动漫科技 bilibili"
# → 找到 B站账号：50万粉丝，日更2部动态漫画

# 第2步：分析内容生产流程
web-search "星辰动漫 招聘"
web-fetch "https://www.zhipin.com/gongsi/星辰动漫科技.html"
# → 招聘信息显示：AI绘画师(ComfyUI)、AE后期、配音师
# → 推断工具链：ComfyUI + AE + 自研配音

# 第3步：发现痛点

# → B站评论区发现多条"角色脸变了""画风不稳定"评论
# → 招聘"AI技术总监"岗位描述中提到"解决角色一致性问题"

# 第4步：企业验证

# → 工商注册正常，ICP备案匹配，注册资本500万，存续状态

# 第5步：判断需求
```
